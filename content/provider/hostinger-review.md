## Performance reality check (what matters for SEO)

What actually impacts SEO for hosting choice:
- **TTFB stability** (not just one speed test)
- **uptime / error rate**
- **cache configuration**
- **resource contention risk** (shared hosting reality)

Hostinger generally performs well for the price, but shared plans can vary under neighbor load. If you care about predictable performance, do this:

### Verification checklist (pass/fail)
Run a 48–72 hour test before committing long-term:

- Test at peak hours (your target market evenings)
- Monitor TTFB and error rate (200 vs 5xx)
- Use a basic load test: 20–50 concurrent requests
- Confirm caching (page cache + object cache for WP) is correctly enabled
- Validate image delivery (CDN or optimized image pipeline)

If TTFB swings wildly or error spikes appear at modest concurrency, upgrade to a higher tier or VPS.

## FAQ

### Is Hostinger good for China traffic?
It depends on routing and where your audience is. For mainland China users, always test:
- peak-hour RTT
- packet loss
- CDN strategy
If China is your core market, consider Hong Kong/Japan/Singapore origin with a CDN.

### Is Hostinger good for WordPress?
Yes for most starter to mid-level WordPress sites, especially if you use caching and keep plugins lean.

### Shared vs VPS: when should I upgrade?
Upgrade when:
- TTFB becomes unstable
- you see 5xx errors at moderate concurrency
- background jobs start impacting page performance

### Is it worth locking in multi-year plans?
Only if you already validated performance and you’re comfortable with the renewal math. Otherwise start shorter.
